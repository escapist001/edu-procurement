# Радар госзаказа

Интерактивный дашборд по рынку учебного оборудования: где в стране закупают, какими процедурами, по каким ценам и кто главные заказчики. Данные — тысяча контрактов из ЕИС, собранных собственным парсером.

**[Открыть дашборд](https://escapist001.github.io/edu-procurement/)**

![Дашборд](preview.png)

## Зачем

Я несколько лет работал в компании — производителе учебного оборудования (АО «А2 Система»), где часть сбыта идёт через госзакупки. Публичные закупки в России — это огромный открытый массив данных, но у ЕИС нет удобного API, а готового среза именно по учебному оборудованию в открытом доступе не найти. Поэтому я собрал данные сам и превратил их в рабочий инструмент.

Дашборд отвечает на вопросы, которые поставщик задаёт себе перед выходом на рынок: в какой регион идти, через какую процедуру заходить, с каким чеком и в какой сезон. Каждый фильтр пересчитывает весь экран — можно проверять свою гипотезу вживую, а не просто смотреть на статичные графики.

## Как это работает

```
zakupki.gov.ru ──parser.py──► contracts.csv ──analysis.py──► rows.json ──► React-дашборд
  (HTML выдачи)   requests+bs4   (1000 строк)     pandas      (все поля)   D3 + Framer Motion
```

1. **`parser.py`** — у ЕИС нет открытого API, но страницы расширенного поиска отдаются обычным HTML. Парсер проходит выдачу по запросу «учебное оборудование» (44-ФЗ и 223-ФЗ) и достаёт из каждой карточки номер, закон, способ, стадию, объект, заказчика, начальную цену, дату и регион.
2. **`analysis.py`** — pandas приводит данные в порядок и выгружает `rows.json`: тысячу контрактов по строкам, с короткими ключами.
3. **Дашборд** (`dashboard/`, React + Vite) — читает `rows.json` и считает все срезы прямо в браузере. Никакого бэкенда: фильтры, скоринг, симулятор и выводы пересчитываются на клиенте. Сборка кладётся в `docs/` для GitHub Pages.

## Что внутри

- **Живые фильтры** — по закону, способу закупки, ценовому сегменту (пороги взяты из 44-ФЗ, а не с потолка) и региону. Рядом всегда видно, сколько контрактов и денег осталось в выборке.
- **Перекрёстная фильтрация** — клик по пузырю региона, столбцу месяца, ячейке матрицы или заказчику включает соответствующий фильтр. Всё, что не попало в выборку, не исчезает, а гаснет: рынок остаётся виден.
- **«Призрак рынка»** — при включённом фильтре за выборкой проступает контур всей совокупности. Видно не просто число, а отклонение от нормы.
- **Автовыводы** — колонка справа пересчитывается под фильтр и говорит по-человечески: какая доля рынка попала в выборку, насколько медиана чека отличается от рыночной, где ниже порог входа, когда пик размещений. Если выборка мала, инструмент честно предупреждает, что выводы ненадёжны.
- **Карта регионов** — пузырьковая диаграмма в четырёх квадрантах: где денег много и чеки крупные, а где мелкий конвейер. Это главный экран — с него понятно, куда ехать в первую очередь.
- **Скоринг «стоит ли заходить»** — композитный балл привлекательности текущего среза из пяти факторов (ёмкость денег, крупность чека, надёжность, открытость, разреженность спроса) с разбивкой вклада каждого.
- **What-if симулятор** — прикидка захода в сегмент: под свою команду и win-rate видно доступный поток лотов, реалистичный выигрыш и выручку.
- **Матрица «способ × чек», сезонность, доля отмен, распределение чеков, Парето заказчиков** — каждый график отвечает на свой вопрос.
- **Флаг мегалотов и нормализация** — единичные контракты-аномалии (≥ 100 млн ₽) помечаются, одной кнопкой убираются, чтобы не искажали средние.
- Ссылка на текущий срез сохраняется в адресе, выборку можно выгрузить в CSV.

## Стек

- **Python 3.12** — `requests` и `BeautifulSoup` для парсера, `pandas` для подготовки данных.
- **React + TypeScript + Vite** — дашборд. Графики нарисованы вручную на SVG с `d3-scale`, анимации — `framer-motion`. Всё считается в браузере, без бэкенда.
- Хостинг — GitHub Pages (сборка в `docs/`).

## Запуск

Данные:

```bash
pip install -r requirements.txt
python parser.py --pages 20   # собрать свежий срез из ЕИС (~1000 записей)
python analysis.py            # подготовить данные → rows.json
```

Дашборд:

```bash
cd dashboard
npm install
npm run dev      # разработка
npm run build    # сборка в ../docs для GitHub Pages
```

## Ограничения выборки

- Дашборд показывает **снимок выборки**, а не всю генеральную совокупность: парсер берёт первую тысячу записей свежей выдачи по ключевому запросу.
- Регион определяется эвристикой по названию заказчика, поэтому возможны неточности и попадания «не определён».
- Запрос идёт с морфологией, так что в выборку попадают и смежные закупки — мебель для классов, канцелярия. Это осознанный охват «околоучебного» рынка.
- Сертификат ЕИС выдан удостоверяющим центром, которого нет в стандартном хранилище, поэтому в парсере отключена проверка TLS. Данные полностью открытые.
